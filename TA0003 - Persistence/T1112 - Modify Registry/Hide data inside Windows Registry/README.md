# Hide data inside Windows Registry - Technical Concepts
Registry is a hierarchical database that stores system configuration settings and installed programs settings. Registry data is structured in a tree format and 
this registry tree can be only 512 levels deep. Each node in the tree is called a key. The following table lists the predefined keys that are used by the system. 
The maximum size of a key name is 255 characters.

| Folder/predefined key | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| HKEY_CURRENT_USER     | Contains the root of the configuration information for the user who is currently logged on. The user's folders, screen colors, and Control Panel settings are stored here. This information is associated with the user's profile. This key is sometimes abbreviated as *HKCU*.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| HKEY_USERS            | Contains all the actively loaded user profiles on the computer. *HKEY_CURRENT_USER* is a subkey of *HKEY_USERS*. *HKEY_USERS* is sometimes abbreviated as *HKU*.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| HKEY_LOCAL_MACHINE    | Contains configuration information particular to the computer (for any user). This key is sometimes abbreviated as *HKLM*.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| HKEY_CLASSES_ROOT     | Is a subkey of `HKEY_LOCAL_MACHINE\Software`. The information that is stored here makes sure that the correct program opens when you open a file by using Windows Explorer. This key is sometimes abbreviated as _HKCR_. Starting with Windows 2000, this information is stored under both the HKEY_LOCAL_MACHINE and HKEY_CURRENT_USER keys. The `HKEY_LOCAL_MACHINE\Software\Classes` key contains default settings that can apply to all users on the local computer. The `HKEY_CURRENT_USER\Software\Classes` key contains settings that override the default settings and apply only to the interactive user. The HKEY_CLASSES_ROOT key provides a view of the registry that merges the information from these two sources. HKEY_CLASSES_ROOT also provides this merged view for programs that are designed for earlier versions of Windows. To change the settings for the interactive user, changes must be made under `HKEY_CURRENT_USER\Software\Classes` instead of under HKEY_CLASSES_ROOT. To change the default settings, changes must be made under `HKEY_LOCAL_MACHINE\Software\Classes`. If you write values to a key under HKEY_CLASSES_ROOT, the system stores the information under `HKEY_LOCAL_MACHINE\Software\Classes`. If you write values to a key under HKEY_CLASSES_ROOT, and the key already exists under `HKEY_CURRENT_USER\Software\Classes`, the system will store the information there instead of under `HKEY_LOCAL_MACHINE\Software\Classes`. |
| HKEY_CURRENT_CONFIG   | Contains information about the hardware profile that is used by the local computer at system startup.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

In addition to the predefined keys, users can be able to create their own keys under these hives. When a key is generated, it can contain multiple values ​​of 
different data types as long as the key does not exceed the 64-kilobyte limit (including all the values ​​of that key). Some of them are quite useful from an 
attacker's perspective.

| Name                    | Data Type     | Description                                                                                                                                                                                                         |
| ----------------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Binary Value            | REG_BINARY    | Raw binary data. Most hardware component information is stored as binary data and is displayed in Registry Editor in hexadecimal format.                                                                            |
| String Value            | REG_SZ        | A fixed-length text string.                                                                                                                                                                                         |
| Multi-String Value      | REG_MULTI_SZ  | A multiple string. Values that contain lists or multiple values in a form that people can read are generally this type. Entries are separated by spaces, commas, or other marks.                                    |
| Expandable String Value | REG_EXPAND_SZ | A variable-length data string. This data type includes variables that are resolved when a program or service uses the data.                                                                                         |
| DWORD Value             | REG_DWORD     | Data represented by a number that is 4 bytes long (a 32-bit integer). Many parameters for device drivers and services are this type and are displayed in Registry Editor in binary, hexadecimal, or decimal format. |
| QWORD Value             | REG_QWORD     | Data represented by a number that is a 64-bit integer. This data is displayed in Registry Editor as a Binary Value and was introduced in Windows 2000.                                                              |

Let's look at some examples to see how these data types can be used for data hiding.

*Example 1 (REG_BINARY):* After gaining access to the target machine, attackers often seek to persist by hiding malware directly within the Windows Registry. 
To achieve this 'fileless' execution, they convert the malicious payload (such as Shellcode or a small DLL) into a raw byte stream. This data is then stored as a 
Binary Value (REG_BINARY). Unlike string-based types, REG_BINARY preserves the integrity of null bytes (0x00) and other non-printable characters essential for 
the malware's execution.

*Example 2 (REG_SZ):* After encrypting data, an attacker may need to store the encryption key or a custom password on the system for later retrieval. 
Since passwords are typically alphanumeric strings, the REG_SZ data type is the most straightforward choice. To avoid manual detection, the attacker might 
obfuscate the password (e.g., using Base64 encoding) and store it under a fake value name like `ProductVersion` or `SerialID` within a legitimate-looking 
registry key.

*Example 3 (REG_MULTI_SZ):* The attackers want their malware to be undetectable by antivirus software. They look for the list of 'Files excluded from virus 
scans' (usually a `REG_MULTI_SZ` value). At this point, they don't just write one, but a list of multiple paths to protect their entire malware:
```
C:\Windows\Temp\malware_1.exe
C:\Windows\Temp\malware_2.exe
C:\Windows\Temp\backdoor.dll
```
Because `REG_MULTI_SZ` allows line breaks, Windows will read and understand these as three separate files that should be ignored during the scan.

*Example 4 (REG_EXPAND_SZ):* After installing the malware on the victim's machine, the attackers want it to automatically start with any user who logs in. Instead 
of using `REG_SZ` with a fixed path for a specific user (such as `C:\Users\Alex\AppData\Microsoft\Windows\Start Menu\Programs\Startup\malware.exe`), he uses 
`REG_EXPAND_SZ` with the data `%AppData%\Microsoft\Windows\Start Menu\Programs\Startup\malware.exe`. Thanks to this, regardless of the user, Windows will 
automatically find that user's folder to run the malware.

*Example 5 (REG_DWORD):* Before transferring malwares to the target machine, the attackers may want to prevent Windows Defender from scanning their malwares, 
they may modify `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Defender\Exclusions\Paths` key, add a value name of the directory which is going to contain 
malwares, the data type is `REG_DWORD` and the data is `0`.

From the examples above, it is clear that an attacker can exploit registry keys to hide malicious data.
