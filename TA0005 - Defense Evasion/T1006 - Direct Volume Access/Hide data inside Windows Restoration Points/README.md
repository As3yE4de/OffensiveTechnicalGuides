# Hide data inside Windows Restoration Points - Technical concepts
VSS (Volume Shadow Copy Service) is a service introduced in Windows Server 2003 to address issues related to data backup and restore and is only 
available on NTFS volumes. These issues are:
- Without VSS, backing up data generated by a running application is impossible; the application must be shut down, and if it's a system application,
it's even more impossible to shut it down midway.
- Even if a backup is performed on a running application without VSS (e.g., via raw disk access), the resulting data remains in a 'Crash-Consistent' state.
This internal inconsistency between the disk and memory often leads to logical data corruption during restoration.
- If the data set is large, it can be difficult to back up all of it at one time.

Windows restoration points which are snapshots of the system's configuration and settings at a specific moment. These restore points can be created 
automatically by the system during significant events, like software installations or updates, or manually before making major changes. Restoration points are 
utilized by System Restore feature on Windows which needs VSS service to work properly. When a restoration point is selected, System Restore feature will 
reverts the system files, registry settings, and installed programs to the state they were in at the time the restore point was created.

Restoration points are not copies of an entire volume, because VSS only operates at the block level. It monitors all blocks on a volume, and when a change 
occurs on a block, VSS takes a snapshot (restore point) of that block before the change happened and stores this read-only snapshot in a hidden volume (diff area). 
We can determine the size of block by using command `fsutil ` on Windows.<br>

A little information about diff area: It can contains the maximum number of software shadow copies for each volume is 512 and 
64 shadow copies that are used by the Shadow Copies of Shared Folders feature. When the diff area reaches the its maximum capacity, the oldest copies will be 
deleted to make rooms for the new ones.

Now, with the ability of VSS service, attackers can leverage System Restore feature to hide malicious data by creating a snapshot and then delete evidences, whenever the attackers want to retrieve data, they simply use `vssadmin` tool which is built-in on Windows systems.
